1. 安裝與啟動

環境需求

  • Python 3.10+
  • Node.js 18+ / yarn
  • (可選) MongoDB — 僅 status_checks API 使用

啟動步驟

# 1. 後端套件
cd backend
pip install -r requirements.txt

# 2. 前端套件
cd ../frontend
yarn install

# 3. 環境變數 (複製 .env.example 並編輯)
cp ../.env.example backend/.env
cp ../.env.example frontend/.env

# 4. 啟動服務 (Supervisor 自動 hot-reload)
sudo supervisorctl start backend frontend

# 5. 瀏覽器開啟
# 前端: http://localhost:3000
# 後端 API: http://localhost:8001/api

環境變數

檔案變數說明
backend/.envMONGO_URLMongoDB 連線字串 (預設 mongodb://localhost:27017)
backend/.envDB_NAME資料庫名稱 (預設 trendy_apparel)
backend/.envCORS_ORIGINS允許的跨域來源 (預設 *)
frontend/.envREACT_APP_BACKEND_URL後端 URL,用於呼叫 MCP API

3. MCP 設定與運作

3.1 MCP 架構

本專題實作 兩個 MCP server,皆以 Python stdio JSON-RPC 通訊,符合 MCP 規格:

  1. trendy-bmi-recommend — 依身高體重 BMI 推薦最多 6 件商品
  2. trendy-product-info — 查詢商品型錄、價格、庫存、預購狀態

3.2 mcp_config.json 完整內容

位於專案根目錄 /mcp_config.json,可在支援 MCP 的客戶端 (如 Claude Desktop、Continue、Cline) 中載入此設定來啟用兩個 server:

{
  "mcpServers": {
    "trendy-bmi-recommend": {
      "command": "python3",
      "args": ["/app/mcp_servers/bmi_recommend.py"],
      "description": "依使用者身高、體重、預算與風格偏好,推薦最多 6 件適合的潮流服飾",
      "tools": [{
        "name": "bmi_recommend",
        "description": "Recommend up to 6 fashion items based on BMI, budget, and style preference",
        "inputSchema": {
          "type": "object",
          "properties": {
            "height": { "type": "number", "description": "Height in cm (100-250)" },
            "weight": { "type": "number", "description": "Weight in kg (20-300)" },
            "budget": { "type": "number" },
            "styles": { "type": "array", "items": { "type": "string" } }
          },
          "required": ["height", "weight"]
        }
      }]
    },
    "trendy-product-info": {
      "command": "python3",
      "args": ["/app/mcp_servers/product_info.py"],
      "description": "查詢商品型錄、價格、庫存、預購狀態",
      "tools": [
        { "name": "get_product",   "description": "Get a product by id" },
        { "name": "list_products", "description": "List products by category" }
      ]
    }
  }
}

3.3 MCP 啟用步驟 (Claude Desktop 範例)

  1. 開啟 Claude Desktop 設定檔 ~/Library/Application Support/Claude/claude_desktop_config.json (macOS) 或 %APPDATA%\Claude\claude_desktop_config.json (Windows)
  2. 將本專案的 mcp_config.json 內 mcpServers 區塊複製貼入
  3. 重啟 Claude Desktop,即可在對話中使用工具bmi_recommend、get_product、list_products

3.4 後端 HTTP Bridge (供前端呼叫)

FastAPI 在 backend/server.py 將 stdio MCP 包裝成 HTTP API,供前端 fetch 呼叫:

EndpointMethod對應 MCP Tool
/api/mcp/bmi-recommendPOSTbmi_recommend
/api/mcp/product/{product_id}GETget_product
/api/mcp/products?category=XGETlist_products

3.5 測試指令

# 1. 測試 BMI 推薦
curl -X POST http://localhost:8001/api/mcp/bmi-recommend \
  -H "Content-Type: application/json" \
  -d '{"height":170,"weight":65,"budget":5000,"styles":["Techwear","Street"]}'

# 預期回應:
# {"bmi":22.5,"category":"正常","sizes":{"top":"M","pant":"32"},
#  "items":["prod_024","prod_001","prod_015","prod_017","prod_020","prod_010"],
#  "note":"...","source":"mcp"}

# 2. 測試商品查詢
curl http://localhost:8001/api/mcp/product/prod_003
curl http://localhost:8001/api/mcp/products?category=jewelry

3.6 失敗時的 Fallback

若 MCP server 啟動失敗或網路中斷,前端 app/js/recommend.js 內 localRecommend() 會接手執行, 邏輯與 MCP server 完全一致,使用者體驗不受影響。回應右上角會顯示灰色 Local Mode 標籤。

5. 常見問題排除

Q1:推薦結果顯示「Local Mode」而非「MCP Live」?

這代表前端無法連到 MCP HTTP bridge。請檢查:

  • backend 服務是否啟動 (supervisorctl status backend)
  • frontend/.env 內 REACT_APP_BACKEND_URL 是否正確
  • 瀏覽器 DevTools → Network 是否有 /api/mcp/bmi-recommend 失敗請求

Q2:商品圖片顯示破圖?

所有 <img> 已加 onerror fallback,單張圖片 404 不影響整體。 若大面積破圖,請檢查 Tailwind CDN 是否能正常載入。

Q3:結帳時提示「資料驗證失敗」?

請確認:電話為 09 開頭 10 碼 / 郵遞區號 3 或 5 碼數字 / Email 格式正確。詳見 .agent/rules/validation.md。

Q4:優惠碼套用失敗?

可能原因:

  • 未達最低消費 (SAVE300 需滿 3000,VIP500 需滿 5000)
  • WELCOME10 已被當前帳號使用過 (一人限用一次)
  • VIP500 與 SAVE300 互斥,不可併用

Q5:如何重置所有資料?

打開瀏覽器 DevTools Console,輸入:

localStorage.removeItem('trendy_apparel_state');
localStorage.removeItem('trendy_users');
location.reload();
