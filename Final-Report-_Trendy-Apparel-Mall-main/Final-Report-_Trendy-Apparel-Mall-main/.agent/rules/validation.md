# Rule — 表單驗證規則 (防呆機制)

為確保訂單資料的正確性，系統在註冊、登入及結帳時實施了嚴格的欄位驗證。

## 驗證標準
1. **姓名 (`name`)**: 至少 2 個字元。
2. **Email (`email`)**: 必須符合標準電子郵件格式 (需含 @ 與 .)。
3. **電話 (`phone`)**: 必須為台灣手機格式 `09xxxxxxxx` (共 10 碼)。
4. **詳細地址 (`address`)**: 至少 5 個字元。
5. **郵遞區號 (`postal`)**: 必須為 3 至 5 碼純數字。
6. **密碼 (`password`)**: 註冊時至少需 6 個字元。

## 觸發時機
- **即時驗證**: 某些欄位在輸入時會檢查格式 (如數量 clamp)。
- **送出前驗證**: 點擊「註冊」、「登入」或「確認下單」時，系統會調用 `Validator` 物件進行全面檢查。

## 程式碼位置
- 驗證對象：`index.html` 中的 `Validator` 常數。
- 呼叫函式：`submitOrder()`、`handleRegister()` 等。
